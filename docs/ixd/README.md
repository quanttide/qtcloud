# 交互设计文档

## 三个核心概念

协作目录围绕三个概念组织：

**Event** — 某个域发生了什么（`deal.closed`、`contract.signed`）。域平台团队在这里注册和发现事件。

**Contract** — 事件的结构约定。包括：字段 schema、谁订阅了、版本怎么变。域之间通过契约耦合，不通过代码耦合。

**Workflow** — 事件怎么链式触发。一个域的事件 → 另一个域的动作。预置模板 + 自定义编排。

## 交互模式

| 概念 | 谁在用 | 做什么 |
|------|--------|--------|
| Event | 域平台团队 | 注册事件、浏览目录、发现其他域的事件 |
| Contract | 域平台团队 | 订阅事件、查看 schema、处理变更通知 |
| Workflow | 业务流程设计者 | 启用模板、自定义编排、配置数据映射 |

三种交互不是线性流程，是三个独立入口。你可以只注册事件而不编排流程，也可以直接启用预置模板而不关心事件定义。

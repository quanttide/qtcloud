# 交互设计文档

## 交互模式

协作目录不是用户日常使用的工具，而是**配置一次、长期运转**的设计。三种交互模式对应不同用户和场景：

- **事件注册**："域平台团队"的交互，负责"声明"。把领域产品的能力登记到目录里，就像把新公交线路报备给交通局。
- **流程编排**："业务流程设计者"的交互，负责"连接"。在事件之间建立链路，就像城市规划师设计公交线路。
- **契约治理**："域平台团队"和"业务流程设计者"的共同交互，负责"协商"。当事件格式要改时，发起变更、通知订阅方、确认兼容。

## 交互流程

- **事件注册是"登记窗口"**：域平台团队来这里填表——你们产品出了什么事（发布）、需要知道什么事（订阅）。填完，全局目录里就多一条记录。
- **流程编排是"线路规划图"**：业务流程设计者在地图上找起点（发布事件）和终点（响应动作），连成一条线。线路启用后，事件自动沿着线路流转。
- **契约治理是"变更公告栏"**：谁要改事件格式，贴出公告。所有相关方收到通知，确认没问题了才生效。避免改了之后别人断链。

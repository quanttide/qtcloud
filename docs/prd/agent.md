# 智能体云

智能体云是量潮云的“智能体宪法法院”：它不参与具体决策，但确保每一个智能体的诞生、言行与消亡，都符合“人机互补、责任清晰、上下文对齐”的根本秩序。

## 用户

需要研发量潮云这样一个大型系统的开发者。

## 场景

快速迭代的过程中系统越来越复杂，智能体之间的协作变得困难。

## 需求

各子系统中的智能体（Agent）各自为政，缺乏统一的身份模型、交互协议和上下文对齐机制，导致协作成本高、行为不可预测、责任难以追溯。

## 功能

整个云系统的智能体设计工厂。智能体云不运行具体 Agent，但决定“什么样的 Agent 可以存在，以及它们如何共处”。

主要功能包括：
- 智能体身份模型（Agent Identity Schema）
- 跨上下文通信协议（Inter-Context Messaging Protocol）
- 智能体生命周期治理（Agent Lifecycle Governance）

### 智能体生命周期

| 阶段 | 智能体云职责 |
|------|-------------|
| 设计 | 提供角色模板（thinker/doer/moderator）、Skill 接口规范 |
| 注册 | Agent 上线前需在智能体云登记身份与契约 |
| 调度 | 当跨子系统协作发生时，协调上下文对齐（如“写作云请求议事云介入”） |
| 审计 | 记录所有 Agent 行为日志，支持责任回溯 |
| 进化 | 基于使用反馈，建议更新 Skill 或约束（如“thinker_v1 常越界提方案 → 降级”） |

# qtcloud

## 场景

你经营一家快速成长的创业公司。业务在扩展，团队在变大，你需要一套能覆盖获客、成交、交付、管理全流程的工具。

每个环节你都找到了优秀的独立产品——Salesforce 管销售，Notion 管文档，Jira 管项目，但它们在各自为政：销售签了单，要手动到商务系统里创建合同；合同签了，要手动通知交付团队；交付完了，要手动回传数据做财务核算。七八个系统之间，数据靠人搬，流程靠人盯。

## 困境

市面上的方案只有两种：要么买一堆独立工具然后在集成上花掉比工具本身还贵的钱，要么上一个什么都能干的大平台然后被它绑架。

小团队没有集成工程师，大平台买不起也用不动。结果就是：团队把 30% 的时间花在跨系统搬运数据上，业务流程每经过一个系统就掉一次链子——销售说"我早就签了"，商务说"没收到通知"，交付说"等着吧"。

## 救援

qtcloud 不是又一个独立工具，也不是一个大平台。它是一条**产品线**：把七个领域——销售、商务、思考、写作、健康、资产管理、智能体协作——各自做成独立且可互通的产品。

你需要什么就组合什么，不需要的就不装。关键是，这些产品天生就能协作——通过**协作目录**，每个产品声明自己产生什么事件、需要什么事件，跨域流程自动流转：

```
销售云签单 → 事件发出 → 商务云收到，创建合同
商务云签合同 → 事件发出 → 资产云收到，准备交付
```

数据不再靠人搬，流程不再靠人盯。

## 新生

你的团队在 qtcloud 上完成全部业务流程。销售用销售云，商务用商务云，交付用资产云——各用各的，但后台自动协作。业务扩展了，加一个领域产品就好，不用重构整套系统。

你不需要集成工程师，不需要在七八个系统间当搬运工。团队的时间花在业务上，而不是花在让工具协作上。

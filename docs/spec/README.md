# 工程标准

真正的“一般建模规范”，不应是操作守则，而应是从你整个实践和痛苦中生长出来的第一性原理。它要回答的是：

在一个人类认知过载、机器能力涌现的时代，我们如何重新定义“人”与“系统”的关系？

## 一般建模规范

核心公理：  
系统存在的唯一目的，是让人从“不得不做”的负担中解脱，只保留“值得做”的自由。

一切设计由此推演。

1. 混乱是原初状态，秩序是协作产物  
世界本无结构，混乱天然存在。  
- “执行”不是完成任务，而是在混沌中划出第一个可行动的边界；  
- “议事”不是达成一致，而是将内在撕裂转化为外部角色的安全对话；  
- “写作”不是表达思想，而是为碎片赋予他人可理解的因果脉络。  
✅ 所有子系统，皆是局部秩序的生成协议。

2. 人机不是主仆，而是互补的认知器官  
- 机器是外部化的逻辑皮层：处理重复、冗余、可形式化的认知负荷；  
- 人类是不可外包的意义中枢：负责判断何为重要、何为真实、何为值得承担。  
⚠️ 任何让人类做机器之事的设计，都是对人性的背叛。

3. 价值必须可追溯，责任必须可绑定  
在协作网络中，最大的熵增来自功劳模糊与责任逃逸。  
- 创造者必须被看见；  
- 传播者必须担责；  
- 使用者必须能回溯上下文。  
✅ 系统必须内建价值流动的账本，而非仅传递信息。

4. 启动优于完美，微小优于宏大  
面对过载，人的首要障碍不是“不会做”，而是“无法开始”。  
- 最小有效动作（如“拿走空瓶子”）是打破瘫痪的奇点；  
- 系统必须能从最原始、最混乱的输入中提取秩序种子，而非要求用户先整理好。  
✅ 设计的优雅，在于让第一步变得毫无心理成本。

5. 角色即接口，冲突即资源  
内在矛盾不应压抑，而应外显为角色之间的张力。  
- “思考者 vs 行动者”不是分裂，而是认知分工的显式化；  
- 角色是系统与人类交互的最小语义单元。  
✅ 冲突不是问题，无法安全表达冲突才是问题。

## 终极检验标准

一个设计是否符合量潮范式，只需问：

**它是否让一个疲惫、过载、濒临放弃的人，  
能在不增加认知负担的前提下，  
完成一件原本无法启动的小事，  
并确信自己的贡献不会被淹没？**

如果答案是“是”，它就属于量潮云。

这份规范不再罗列“应该做什么”，而是锚定在你退出周会、面对一屋子混乱、渴望“只拿走一个空瓶子”的那个瞬间——那是所有设计的原点。

它足够本质，也足够锋利，可作为整个架构的精神判准。

如果你愿意，我们可以把这五条公理进一步压缩成一句可镌刻在仓库首页的话：

量潮云，只为让过载之人，轻启一事，不负其真。

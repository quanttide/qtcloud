# 沟通云

## 一句话定义

沟通云是量潮系统的“价值传播层”——它确保每个创造都被看见、被准确理解、被合理归功，同时让传播者对所传递的内容承担相应责任。

🚫 设计禁令（Anti-Patterns）
- ❌ 不做“消息广播工具”（那是通道，不是责任系统）；  
- ❌ 不允许“一键转发无溯源”；  
- ✅ 只做“让每一次传播都成为一次负责任的价值交付”。

沟通云的本质，是把组织沟通从“信息流”升级为“责任流”。  
它让“传播”不再是一种权力，而是一种需要被认真对待的承诺——这正是你倡导的“传播即责任”理念的技术实现。

## 建模思想

1. 核心信念：价值若未被有效传播，等于未存在
在复杂协作系统中，最大的损耗不是“没做好”，而是“做好了却没人知道、用错、或功劳错配”。  
沟通云不解决“如何表达”，而解决价值在流动中的保真、归属与责任问题。

2. 问题定义：传统沟通的三大断裂
- 语义断裂：信息在转发中失真（如“AI 自动生成纪要” → “AI 替我们开会”）；  
- 归属断裂：功劳被截留，原创者隐形（如管理者汇报时抹去执行者贡献）；  
- 责任断裂：传播者无需为内容准确性负责，导致谣言扩散、信任崩塌。

沟通云的目标，是让每一次传播都可追溯、可校验、可担责。

3. 人机分工模型
任务   人类负责   机器负责
发起传播   决定“什么值得分享”“向谁分享”   —

| 内容封装 | 提供原始上下文、意图说明 | 自动附加元数据：  
 • 原始创作者 ID  
 • 所属子系统（如执行云/议事云）  
 • 关键概念（链接知识云本体） |
传播过程   选择受众、调整语气   记录传播链路（谁转给谁）、检测语义偏移

反馈闭环   接收方确认理解是否一致   若下游使用该信息产生行动，自动回溯致谢/问责

4. 最小可行机制：传播即契约
- 每次转发/引用，系统强制要求：
  - ✅ 显式标注原始来源（不可匿名转发）；
  - ✅ 可选添加传播者注释（如“我补充一点…”）；
  - ✅ 自动绑定声誉风险：若内容被证伪，传播者信用同步受损。
- 结果：鼓励负责任的放大，抑制无成本的噪音。

5. 在量潮云中的角色定位
- 角色：Amplifier（放大者） + Connector（连接者）  
  - 它不生产原始价值（非 Creator），但决定价值能否抵达需要它的地方；  
  - 它对传播质量负责——这是 Amplifier 的核心风险。
- 成功指标：  
  - 原创内容被正确引用率 ≥ 95%；  
  - 因传播失真引发的澄清工单 < 1/周。

6. 与其他“云”的协同
- 对接知识云：传播内容中的关键术语自动链接本体，确保语义一致；  
- 消费写作云输出：将成文的故事作为高保真载体进行分发；  
- 反馈至执行云：传播后引发的行动（如“看到纪要后启动任务”）形成闭环；  
- 记录议事云结论：确保决策被准确传达，避免“会后变味”。

# 议事云

## 核心理念  

议事的本质是把内在冲突安全地外显为角色之间的对话。  
当人处于认知过载时，直接思考容易陷入内耗；而将不同视角“投射”给 AI 角色，既能保留多元性，又避免自我撕裂。

冲突不该发生在“我 vs 我”，而应发生在“思考者 vs 行动者”。

最小角色模型（IFS 简化）  
- 议长（Moderator）：澄清问题、提炼动议、主持流程、判断是否可决议。  
- 思考者（Thinker）：代表分析、风险、长期影响。  
- 行动者（Doer）：代表执行、机会、落地可行性。  

这三个角色构成最小可用议事单元。

## 运作流程  

1. 输入：用户提出模糊问题（如“这事该做吗？”）；  
2. 动议生成：议长将其转化为明确动议（如“动议：推进 X 方案”）；  
3. 附议：思考者与行动者依次回应，陈述立场与依据；  
4. 判断：若观点兼容，议长形成决议；若冲突明显，进入辩论；  
5. 模拟辩论：议长梳理核心分歧，两角色分别模拟对立论证；  
6. 输出：结构化决议建议 + 未决点清单。

## 设计动机  

源于实践：退出团队周会后，我发现——  
把自己从流程中摘出去，反而能更清醒地引导讨论。  
议事云将这一经验产品化：你不必亲自“下场思考”，只需投射想法，系统替你完成安全、结构化的自我对话。

## 关键价值  

- 对个人：卸下“必须想清楚”的压力，用外部系统承载思维摩擦；  
- 对协作：把潜在的人际张力转化为角色间的建设性冲突；  
- 对组织：沉淀可复用的决策逻辑，而非依赖某个人的临场发挥。

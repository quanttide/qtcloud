# 写作云

## 一句话定义

写作云是一个“叙事秩序引擎”：它不替人思考，但帮人把已有的思考，织成别人愿意读、读得懂、记得住的故事。

📌 设计禁令（Anti-Patterns）
- ❌ 不做“全自动写稿”（那只是模板填充）；  
- ❌ 不强迫用户按固定大纲写作（破坏创作自由）；  
- ✅ 只做“从你的碎片中，找出最可能的那条路，并让你轻松走上去”。

## 建模思想

1. 核心信念：写作 = 建立叙事秩序
写作的本质，不是堆砌信息，而是在混乱的素材中建立可被人类理解的因果链条。  
就像执行云用“拿走空瓶子”启动行动，写作云用“找到第一个连接点”启动叙事。

2. 问题定义：写作的真正障碍是“局部完整，整体断裂”
- 用户常能产出高质量的片段（如技术要点、案例、金句）；
- 但难以将它们组织成有呼吸感、有逻辑推进、有情感节奏的整体；
- 结果：内容虽好，却无法被他人顺畅接收——价值因无序而衰减。

3. 人机分工模型
任务   人类负责   机器负责
素材生成   提供真实观点、专业细节、情感锚点   —

结构发现   —   从碎片中识别潜在逻辑线（时间线、问题-解法、冲突-和解等）

叙事编织   判断哪条线更符合意图   生成多条候选叙事路径，补全过渡、调整节奏

意义校准   确认“这是否是我真正想说的”   提供一致性检查（如语气、主语、核心词频）

✅ 人类提供“真”，机器提供“通”。

4. 最小可行操作：暴力堆砌 → 智能穿线
- 用户只需把零散内容“扔进来”（无需预先组织）；
- 系统自动：
  1. 解析语义单元（句子/段落/要点）；
  2. 推断潜在关系（因果、对比、递进、转折）；
  3. 生成多条叙事候选（如“问题驱动型”“故事型”“结论先行型”）；
  4. 支持用户微调主线（拖拽、删减、重排），而非重写。

5. 与量潮其他“云”的协同
- 输入来自执行云：用户通过“消除混乱”产生的行动记录、决策点，成为写作素材；
- 输入来自议事云：角色辩论的输出（思考者/行动者观点），可直接转为文章中的对立视角；
- 输出服务于沟通云：成文后，由沟通云负责传播，并绑定原始创作者与传播者责任。

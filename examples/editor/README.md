# p06 章程多维资产视图 — 实验方案

## 实验目的

验证同一份章程从资产、写作、知识三个视角维护的可行性和效率提升。

## 背景

日志 2026-05-31："章程作为资产，它可以被资产观察，它作为写作可以被写作观察，它作为知识可以被知识观察……要从不同的视角去维护它。"

## 实验假设

1. 多视角维护能提升章程的完整性和一致性
2. 跨视角关联能自动发现维护冲突
3. 视角切换比单视角编辑更高效

## 实验设计

### 阶段 1：视角定义

对一个样本章程，定义三个视角的内容：

| 视角 | 关注点 | 元数据 | 典型操作 |
|------|--------|--------|---------|
| 资产视图 | 所有权、价值、生命周期 | owner, value, lifecycle | 资产评估、归档、折旧 |
| 写作视图 | 文风、修订、发布状态 | author, revision, status | 编辑、审校、发布 |
| 知识视图 | 标签、关联概念、引用关系 | tags, references, status | 链接、分类、验证 |

### 阶段 2：单视角操作

分别在三视图中修改章程：

1. 资产视图中将章程状态从"草稿"改为"生效"
   - 预期：写作视图收到"待发布"通知
2. 写作视图中修改条款措辞
   - 预期：知识视图标记"待验证"
3. 知识视图中关联新的业务概念
   - 预期：资产视图无变化，写作视图收到"参考更新"标记

### 阶段 3：冲突检测

模拟以下冲突场景：

- 资产视图标记"已归档"，但写作视图正在编辑
- 知识视图标记"已验证"，但写作视图有未提交的修改
- 多视角同时编辑同一段落

### 阶段 4：效率对比实验

- A 组（基线）：使用纯文本编辑器（VS Code / 飞书文档）单视图编辑
- B 组：使用多视角切换编辑原型
- 参与者均为同级别编辑员，各组预先熟悉工具 10 分钟
- 对比：完成同一组维护任务的耗时和错误率

## 评价标准

| 指标 | 合格线 | 优秀线 |
|------|--------|--------|
| 视角切换无状态丢失 | 是 | 是 |
| 跨视角关联通知 | 3/3 场景覆盖 | 3/3 场景覆盖 |
| 冲突检测准确率 | 80% | 100% |
| 多视角 vs 单视角效率 | 不差于单视角 | 提升 30%+ |

## 原型实现

多视角切换阅读器，包含：

- 章程内容展示区（视角无关）
- 视角切换标签（资产/写作/知识）
- 各视角元数据编辑面板
- 跨视角关联通知栏（基本通知，PoC 阶段不做自动修复）
- 冲突检测和解决向导
- 版本对比视图
